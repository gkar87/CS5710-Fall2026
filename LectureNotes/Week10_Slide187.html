Gated Recurrent Unit (GRU)

What is GRU?

Gated Recurrent Unit is a simplified version of LSTM that combines the forget and input gates into a single "update gate" and merges cell state and hidden state.

Key Innovation: GRU achieves similar performance to LSTM with fewer parameters and simpler computation.

GRU Equations
Update Gate
z_t = σ(W_z · [h_{t-1}, x_t])
Reset Gate
r_t = σ(W_r · [h_{t-1}, x_t])
Candidate State
h̃_t = tanh(W · [r_t ⊙ h_{t-1}, x_t])
Final Update
h_t = (1-z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t

GRU vs LSTM

Why Create GRU?

  • LSTM works well but is computationally expensive
  • Many applications don't need LSTM's full complexity
  • Simpler models train faster and are easier to tune
  • Fewer parameters reduce overfitting risk

Key Simplifications:

  • 2 gates instead of 3: Update + Reset vs Forget + Input + Output
  • No separate cell state: Uses only hidden state
  • Direct state updates: No separate memory pathway
  • Fewer weight matrices: Reduced parameter count

Performance Trade-offs:

  • Speed: 20-30% faster training
  • Memory: Lower memory usage
  • Accuracy: Similar on most tasks
  • Generalization: Often better on smaller datasets

Architecture Comparison

🧠 LSTM (3 Gates)
  • 🚫 Forget Gate (what to remove)
  • 📥 Input Gate (what to add)
  • 📤 Output Gate (what to output)
  • 🛣️ Separate cell state pathway
  • ⚙️ Complex internal mechanics
  • 🎯 Best for complex, long sequences
⚡ GRU (2 Gates)
  • 🔄 Update Gate (how much to update)
  • 🔁 Reset Gate (what past info to use)
  • 🚀 Single state (no cell state)
  • ⚡ Simpler, faster computation
  • 📦 Fewer parameters to tune
  • 🎯 Great for most practical tasks

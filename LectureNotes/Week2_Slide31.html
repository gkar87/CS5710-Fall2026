Polynomial Regression

Beyond Linear Relationships

Polynomial Regression extends linear regression to capture non-linear relationships by adding polynomial terms of the input features. It remains linear in the coefficients but models curved patterns in the data.
  • Non-Linear Patterns
    Captures curved relationships like growth curves, U-shaped patterns, and saturation effects that straight lines cannot model. Essential for real-world data where relationships are rarely perfectly linear.
  • Feature Engineering
    Creates new features by raising existing features to different powers (x², x³, etc.). Transforms the feature space to enable linear methods to fit non-linear patterns.
  • Still Linear in Parameters
    Despite using non-linear features, the model remains linear in the coefficients (β values). This means we can still use standard linear regression optimization techniques like gradient descent or the normal equation.
  • Overfitting Risk
    Higher-degree polynomials can fit training data perfectly but often fail on new data. A degree 10 polynomial with 11 data points fits perfectly but generalizes poorly. Use cross-validation and regularization to find the right balance.

Mathematical Foundation

General Polynomial Form:
y = β₀ + β₁x + β₂x² + β₃x³ + ... + βₙxⁿ
Where:
• n = degree of polynomial (hyperparameter to choose)
• β₀, β₁, ..., βₙ = coefficients learned by the model
• Each x^i creates a new engineered feature
• Optimization is identical to linear regression (still convex!)
Key Insight:
We transform the feature space (not the model) to make non-linear problems solvable with linear methods!

Common Polynomial Degrees

1
Linear
y = β₀ + β₁x
2
Quadratic
y = β₀ + β₁x + β₂x²
3
Cubic
y = β₀ + β₁x + β₂x² + β₃x³

Actor-Critic Methods

Architecture Overview

Actor-Critic combines policy gradient (Actor) with value function approximation (Critic) for lower variance and better stability.

🎭 Actor

π(a|s; θ)
Selects Actions
⟷

🎯 Critic

V(s; w)
Evaluates States
Key Insight:
Critic reduces variance of policy gradient
Actor provides better exploration than greedy

Core Components

  • 📊 Advantage Function
    A(s,a) = Q(s,a) - V(s)
  • 🔄 TD Error
    δ = r + γV(s') - V(s)
  • 🎲 Policy Update
    θ ← θ + α∇θ log π(a|s)·A(s,a)
  • 📈 Value Update
    w ← w + β·δ·∇wV(s;w)
  • ⚖️ Two Timescales
    Critic learns faster than Actor

Actor-Critic Algorithm

Actor-Critic with TD(0):

# Initialize actor π(a|s;θ) and critic V(s;w)
Initialize θ and w arbitrarily

Repeat for each episode:
  Initialize state S
  I ← 1 # Discount factor accumulator
  
  While S is not terminal:
    # Actor: Select action
    A ~ π(·|S;θ)
    Take action A, observe R, S'
    
    # Critic: Compute TD error
    δ ← R + γV(S';w) - V(S;w)
    
    # Update critic
    w ← w + β·δ·∇wV(S;w)
    
    # Update actor
    θ ← θ + α·I·δ·∇θ log π(A|S;θ)
    
    I ← γI
    S ← S'

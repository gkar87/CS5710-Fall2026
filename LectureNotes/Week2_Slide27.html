Stochastic Gradient Descent (SGD)

What is SGD?

Stochastic Gradient Descent updates parameters using the gradient computed from a single randomly selected training example at each iteration, rather than the entire dataset.
θ = θ - α · ∇ℓ(hθ(x⁽ⁱ⁾), y⁽ⁱ⁾)

where (x⁽ⁱ⁾, y⁽ⁱ⁾) is randomly sampled
  • Updates parameters after every single sample
  • Much faster iterations than batch gradient descent
  • Noisy gradient estimates with high variance
  • Can escape local minima due to noise
  • Enables online learning from streaming data
  • Memory efficient - processes one sample at a time

SGD Algorithm

  • Step 1: Initialize
    Set θ to random values, choose learning rate α
  • Step 2: Shuffle Data
    Randomly shuffle training examples for each epoch
  • Step 3: Select Sample
    Pick one training example (x⁽ⁱ⁾, y⁽ⁱ⁾) randomly
  • Step 4: Compute Gradient
    Calculate ∇ℓ(hθ(x⁽ⁱ⁾), y⁽ⁱ⁾) for this sample only
  • Step 5: Update Parameters
    θ = θ - α · gradient (immediate update!)
  • Step 6: Repeat
    Continue until convergence or max iterations

🔄 Gradient Descent Variants Comparison

Batch GD
Uses all m samples Stable convergence Slow for large datasets High memory usage Exact gradients
SGD
Uses 1 sample Noisy but fast Escapes local minima Memory efficient Online learning
Mini-batch GD
Uses k samples Balance of both Vectorization benefits Moderate memory Most popular choice
💡 SGD Trade-off:
Speed and memory efficiency vs. stability and convergence smoothness. The noise in SGD can be both a blessing (escapes local minima) and a curse (never fully converges).

🎯 Convolutional Layers

  • Local Connectivity: Each neuron connects to local region
  • Parameter Sharing: Same filter applied across input
  • Translation Equivariance: Output shifts with input
  • Hierarchical Features: Low to high level patterns
  • Multiple Filters: Detect different features simultaneously

🏊 Pooling Layers

  • Spatial Reduction: Reduces feature map dimensions
  • Translation Invariance: Small shifts don't affect output
  • Computational Efficiency: Fewer parameters to process
  • Overfitting Control: Acts as regularization
  • Receptive Field: Increases effective field of view

🔗 Fully Connected Layers

  • Global Integration: Combines all spatial features
  • Classification: Maps features to class probabilities
  • High Parameters: Most parameters often here
  • Flatten Operation: Converts 3D to 1D representation
  • Dropout: Often used to prevent overfitting

⚡ Design Principles

  • Depth vs Width: Balance network depth and filter count
  • Receptive Field: Ensure adequate spatial coverage
  • Skip Connections: Help gradient flow in deep networks
  • Batch Normalization: Stabilize training process
  • Data Augmentation: Improve generalization

🎬 Interactive Architecture Visualization

💡 Key Architecture Insights

  • Feature Hierarchy: Early layers detect edges/textures, deeper layers detect objects/concepts
  • Spatial Resolution: Decreases through pooling while feature complexity increases
  • Parameter Efficiency: Convolution uses far fewer parameters than fully connected layers
  • Translation Invariance: CNNs can recognize objects regardless of position in image

📥 Input Layer Details

Input Layer represents the raw image data entering the CNN:

  • Dimensions: Height × Width × Channels
  • Grayscale: H × W × 1 (single channel)
  • RGB Images: H × W × 3 (red, green, blue channels)
  • Preprocessing: Normalization, mean subtraction, data augmentation

🎨 Input Representation

📊 Common Input Sizes

  • CIFAR-10: 32×32×3 (small natural images)
  • ImageNet: 224×224×3 (standard benchmark)
  • High Resolution: 512×512×3 or larger for detailed tasks
×

🎯 Convolutional Layers Deep Dive

Convolutional layers are the heart of CNNs, performing feature extraction:

🎬 Convolution Animation

🔧 Key Hyperparameters

  • Filter Size: Usually 3×3, 5×5, or 7×7 (3×3 most common)
  • Stride: Step size of filter movement (usually 1)
  • Padding: Zero-padding to control output size
  • Number of Filters: Determines output depth/channels

✨ Core Properties

  • Local Connectivity: Each neuron only connects to a small spatial region
  • Parameter Sharing: Same filter weights used across entire input
  • Translation Equivariance: If input shifts, output shifts correspondingly
  • Feature Detection: Learn to detect edges, textures, patterns automatically
×

⚡ CNN Design Principles

🏗️ Complete Architecture Flow

🎯 Architectural Guidelines

  • Depth Strategy: Deeper networks learn more complex representations
  • Width Considerations: More filters per layer capture diverse features
  • Receptive Field: Should cover relevant spatial context for the task
  • Skip Connections: Help gradients flow in very deep networks (ResNet)

🚀 Modern Improvements

  • Batch Normalization: Stabilizes training and speeds convergence
  • Dropout: Prevents overfitting by randomly zeroing neurons
  • Global Average Pooling: Reduces parameters vs fully connected
  • Depthwise Convolutions: More efficient parameter usage

📈 Feature Learning Hierarchy

  • Layer 1-2: Edges, corners, basic geometric shapes
  • Layer 3-4: Textures, simple patterns, basic shapes
  • Layer 5+: Complex objects, scenes, high-level concepts
×

⚡ Activation Functions

Activation functions introduce non-linearity, enabling networks to learn complex patterns:

  • ReLU: f(x) = max(0, x) - Most popular, simple and effective
  • Leaky ReLU: f(x) = max(0.01x, x) - Handles negative values better
  • ELU: Exponential Linear Unit with smooth negative part
  • Swish/SiLU: f(x) = x × sigmoid(x) - Self-gated activation
×

🏊 Pooling Operations

Pooling reduces spatial dimensions while preserving important information:

  • Max Pooling: Takes maximum value in pooling window
  • Average Pooling: Takes average value in pooling window
  • Global Pooling: Reduces entire feature map to single value
  • Typical Size: 2×2 window with stride 2 (non-overlapping)
×

🔗 Fully Connected Layers

Fully connected layers perform the final classification step:

  • Flattening: Convert 3D feature maps to 1D vector
  • Dense Connections: Every input connected to every output
  • High Parameters: Often contains most network parameters
  • Softmax Output: Converts raw scores to class probabilities
  • Modern Alternative: Global Average Pooling + small FC layer

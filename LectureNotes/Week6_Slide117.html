Clustering Evaluation Metrics

E
External Metrics
Compare clustering results against known ground truth labels. Useful when you have labeled data for validation.
  • ✓ Adjusted Rand Index (ARI)
  • ✓ Normalized Mutual Information (NMI)
  • ✓ Homogeneity & Completeness
  • ✓ V-measure
I
Internal Metrics
Evaluate clustering quality using only the data and cluster assignments. No ground truth required.
  • ✓ Silhouette Score
  • ✓ Davies-Bouldin Index
  • ✓ Calinski-Harabasz Index
  • ✓ Inertia/Within-cluster Sum of Squares

📊 Key Evaluation Metrics

Silhouette Score
s = (b - a) / max(a, b)
Measures how similar a point is to its own cluster compared to other clusters. Higher values indicate better clustering.
Range: [-1, 1], Higher is better
Adjusted Rand Index
ARI = (RI - Expected_RI) / (max(RI) - Expected_RI)
Measures similarity between predicted and true clustering, adjusted for chance. Handles different cluster sizes.
Range: [-1, 1], Higher is better, 0 = random
Davies-Bouldin Index
DB = (1/k) Σ max((σᵢ + σⱼ)/d(cᵢ,cⱼ))
Ratio of within-cluster to between-cluster distances. Lower values indicate better clustering with compact, well-separated clusters.
Range: [0, ∞), Lower is better
Calinski-Harabasz Index
CH = (SSB/(k-1)) / (SSW/(n-k))
Ratio of between-cluster dispersion to within-cluster dispersion. Higher values indicate better defined clusters.
Range: [0, ∞), Higher is better
Normalized Mutual Information
NMI = 2 * MI(U,V) / (H(U) + H(V))
Measures mutual information between predicted and true clustering, normalized by entropy. Robust to cluster imbalance.
Range: [0, 1], Higher is better
V-measure
V = 2 * (h * c) / (h + c)
Harmonic mean of homogeneity and completeness. Balances cluster purity with completeness of cluster assignment.
Range: [0, 1], Higher is better

🔍 When to Use Each Metric

Metric Best For Advantages Limitations
Silhouette Score General evaluation, interpretable results Easy to interpret, works with any distance metric Biased towards convex clusters
ARI Comparing with ground truth Adjusted for chance, handles different cluster sizes Requires true labels
Davies-Bouldin Compact, well-separated clusters Based on cluster geometry, no ground truth needed Assumes spherical clusters
Calinski-Harabasz Determining optimal number of clusters Fast computation, good for k selection Biased towards spherical clusters
NMI Information-theoretic evaluation Robust to cluster imbalance Requires ground truth labels

Regression Evaluation Metrics

🎯 R² (R-Squared)

R² = 1 - (SS_res / SS_tot)
Coefficient of Determination - Measures the proportion of variance in the dependent variable explained by the model.
Range: (-∞, 1] | Perfect: 1.0 | Baseline: 0.0
✓ Easy to interpret
✓ Scale-independent
✗ Can be negative for poor models

📊 MSE (Mean Squared Error)

MSE = (1/n)∑(yᵢ - ŷᵢ)²
Mean Squared Error - Average of squared differences between predicted and actual values. Heavily penalizes large errors.
Range: [0, +∞) | Perfect: 0.0 | Lower is better
✓ Differentiable (good for optimization)
✓ Penalizes large errors heavily
✗ Same units as y² (hard to interpret)

📈 MAE (Mean Absolute Error)

MAE = (1/n)∑|yᵢ - ŷᵢ|
Mean Absolute Error - Average of absolute differences. More robust to outliers than MSE.
Range: [0, +∞) | Perfect: 0.0 | Lower is better
✓ Same units as y (interpretable)
✓ Robust to outliers
✗ Not differentiable at zero

📎 RMSE (Root Mean Squared Error)

RMSE = √[(1/n)∑(yᵢ - ŷᵢ)²]
Root Mean Squared Error - Square root of MSE. Returns error to original scale, making it more interpretable.
Range: [0, +∞) | Perfect: 0.0 | Lower is better
✓ Same units as y (interpretable)
✓ Penalizes large errors
✗ Sensitive to outliers

🔄 Metric Comparison Guide

Scenario Best Metric Why? Interpretation
Model Comparison R² Scale-independent, easy to compare across different datasets Higher R² = better model fit
Business Reporting RMSE or MAE Same units as target variable, easier for stakeholders to understand "Model is off by ±X units on average"
Outlier-Heavy Data MAE Robust to outliers, doesn't get skewed by extreme values Typical absolute deviation
Large Errors are Critical MSE or RMSE Squares heavily penalize large prediction errors Emphasizes minimizing worst-case errors
Model Optimization MSE Differentiable everywhere, works well with gradient-based methods Standard loss function for regression
Poor Model Performance R² (can be negative!) R² can be misleading when model performs worse than baseline Use MSE/RMSE/MAE instead

💡 Pro Tips

  • Always use multiple metrics for comprehensive evaluation
  • Consider the business context when choosing primary metric
  • Cross-validate your evaluation to ensure robustness
  • Plot residuals to check for patterns your metrics might miss

⚠️ Common Pitfalls

  • Don't rely on a single metric
  • R² can be negative for very poor models
  • MSE units are squared (harder to interpret directly)
  • Metrics can be misleading with skewed target distributions

DBSCAN Parameters (eps, min_samples)

ε
Epsilon (ε)
The maximum distance between two points for them to be considered neighbors. It defines the radius of the neighborhood around each point.
  • 🔵 Too small: Many points become noise, over-fragmentation
  • 🔴 Too large: Distinct clusters merge together
  • ⚖️ Just right: Natural cluster boundaries preserved
  • 📏 Units matter: Should match your data scale
  • 🎯 K-distance graph helps find optimal value
M
Min Samples
The minimum number of points required in a neighborhood (including the center point) for a point to be classified as a core point.
  • 🔽 Too small: More noise classified as clusters
  • 🔼 Too large: Smaller clusters become noise
  • 📊 Rule of thumb: 2 × dimensionality
  • 🎲 Higher dimensions need higher min_samples
  • ⚡ Minimum value should be 3 for meaningful clusters

🎯 Parameter Tuning Guidelines

K-Distance Graph Method
Plot the distance to the k-th nearest neighbor for each point (sorted). The "elbow" or steep change indicates optimal ε. Use k = min_samples - 1.
Domain Knowledge Approach
Use your understanding of the data. If clustering cities, ε might be geographical distance. For customer data, consider meaningful business metrics.
Grid Search Validation
Try multiple parameter combinations and use silhouette score or other clustering metrics to evaluate quality. Cross-validate results.
Dimensionality Considerations
Higher dimensions require larger min_samples (rule: 2×D) and careful ε selection due to curse of dimensionality. Consider dimensionality reduction first.

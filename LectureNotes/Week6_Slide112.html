Core Points, Border Points, and Noise

C
Core Points
A point with at least minPts neighbors within radius ε (including itself). These points are at the center of dense regions.
  • 🎯 Seeds of new clusters
  • 🌟 High local density
  • 🔗 Can directly reach other points
  • ✅ Always assigned to clusters
B
Border Points
A point with fewer than minPts neighbors within radius ε, but within the neighborhood of at least one core point.
  • 🏷️ On cluster boundaries
  • 📍 Medium local density
  • ⬅️ Reachable from core points
  • 🔄 May belong to multiple clusters
N
Noise Points
A point with fewer than minPts neighbors within radius ε and not within any core point's neighborhood. These are outliers.
  • 🚫 Not assigned to any cluster
  • 📉 Very low local density
  • ⚡ Potential anomalies/outliers
  • 🔍 Isolated from dense regions

🔍 How DBSCAN Determines Point Types

Step 1: Count Neighbors
For each point, count how many points (including itself) are within distance ε. This determines the local density around each point.
Step 2: Classify as Core or Non-Core
If neighbor count ≥ minPts, the point is a core point. Otherwise, it's either a border point or noise (determined in next steps).
Step 3: Find Density Reachability
Non-core points within ε distance of any core point become border points. They're "reachable" from dense regions.
Step 4: Identify Noise
Points that are neither core points nor reachable from any core point are classified as noise/outliers.

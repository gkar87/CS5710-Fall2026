Learning Rate Scheduling

Why Schedule Learning Rates?

Learning rate scheduling dynamically adjusts the learning rate during training to improve convergence and final model performance. Starting with a high learning rate allows for rapid initial progress, while decreasing it over time enables fine-tuning.

Key Benefits:

  • Escape local minima early in training
  • Fine-tune weights as training progresses
  • Improve convergence stability
  • Achieve better final performance
📉 Step Decay
Reduces learning rate by a factor at specific intervals
lr = initial_lr * decay_rate^(epoch / drop_rate)
initial_lr: Starting learning rate
decay_rate: Multiplicative factor (e.g., 0.5)
drop_rate: Epochs between drops
📊 Exponential Decay
Smoothly decreases learning rate exponentially
lr = initial_lr * exp(-decay_rate * epoch)
initial_lr: Starting learning rate
decay_rate: Decay constant
Smooth: Continuous decay
🌊 Cosine Annealing
Follows a cosine curve from initial to minimum value
lr = lr_min + 0.5 * (lr_max - lr_min) * (1 + cos(π * epoch / max_epochs))
lr_max: Maximum learning rate
lr_min: Minimum learning rate
Smooth: Gradual transition
🔥 Warm Restarts
Periodically resets learning rate to escape local minima
lr = lr_min + 0.5 * (lr_max - lr_min) * (1 + cos(π * t_cur / t_i))
t_cur: Current iteration in cycle
t_i: Cycle length
Restarts: Multiple cycles

Best Practices:

  • Start with a learning rate finder to determine optimal initial rate
  • Use cosine annealing for most deep learning tasks
  • Consider warm restarts for very long training runs
  • Monitor validation loss to adjust schedule parameters
  • Combine with adaptive optimizers (Adam, RMSprop) carefully

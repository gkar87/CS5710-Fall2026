Text Classification with Naive Bayes

From Text to Features

Text Classification using Naive Bayes requires converting raw text into numerical features that the algorithm can process.
📝 Bag of Words (BoW)
Count frequency of each word - ignores order, focuses on presence/absence
📊 TF-IDF
Term Frequency × Inverse Document Frequency - weights important words higher
✅ Binary Features
Simply presence (1) or absence (0) of words - works well with Naive Bayes
🔗 N-grams
Sequences of N words - captures some context (bigrams: "not good", "very happy")

Text Processing Pipeline

Text Preprocessing
Lowercase, remove punctuation, handle special characters, tokenization
Remove Stop Words
Filter out common words like "the", "and", "is" that don't carry much meaning
Stemming/Lemmatization
Reduce words to root forms: "running" → "run", "better" → "good"
Vectorization
Convert text to numerical feature vectors using BoW, TF-IDF, or other methods
Model Training
Calculate word probabilities P(word|class) for each class from training data

🔬 Multinomial Naive Bayes Formulas

Multinomial Probability
P(word|class) = (count(word, class) + α) / (total_words_in_class + α × vocab_size)
α = smoothing parameter (usually 1 for Laplace smoothing)
Text Classification
P(class|text) ∝ P(class) × ∏ P(word|class)^count(word)
Product over all words in vocabulary, raised to their count in the document
Log-Space Calculation
log P(class|text) = log P(class) + Σ count(word) × log P(word|class)
Prevents numerical underflow when multiplying many small probabilities
TF-IDF Weighting
TF-IDF = (word_count / doc_length) × log(num_docs / docs_containing_word)
Alternative to simple word counts - emphasizes distinctive words

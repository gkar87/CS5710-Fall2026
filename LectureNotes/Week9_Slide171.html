🌟 VGG-16 Architecture (Most Popular)

Configuration D: 16 weight layers (13 conv + 3 FC)
# VGG-16 Complete Architecture def make_vgg16(): layers = [] in_channels = 3 # Configuration: [64, 64, 'M', 128, 128, 'M', 256, 256, 256, 'M', # 512, 512, 512, 'M', 512, 512, 512, 'M'] cfg = [64, 64, 'M', 128, 128, 'M', 256, 256, 256, 'M', 512, 512, 512, 'M', 512, 512, 512, 'M'] for v in cfg: if v == 'M': layers += [nn.MaxPool2d(kernel_size=2, stride=2)] else: layers += [nn.Conv2d(in_channels, v, kernel_size=3, padding=1), nn.BatchNorm2d(v), nn.ReLU(inplace=True)] in_channels = v return nn.Sequential(*layers) # Feature dimensions through network: # Input: 224×224×3 # Block1: 224×224×64 → 112×112×64 # Block2: 112×112×128 → 56×56×128 # Block3: 56×56×256 → 28×28×256 # Block4: 28×28×512 → 14×14×512 # Block5: 14×14×512 → 7×7×512 # FC: 7×7×512 = 25088 → 4096 → 4096 → 1000
Why VGG-16 is Special:
• Most widely used for transfer learning
• Excellent feature extractor
• Pre-trained weights readily available
• Good trade-off between depth and computation

🔥 VGG-19 Architecture

Configuration E: 19 weight layers (16 conv + 3 FC)
VGG-19 adds 3 more convolutional layers compared to VGG-16: • Extra conv layer in blocks 3, 4, and 5 • Marginal improvement over VGG-16 (~0.1% better) • More computational cost for minimal gain
# VGG-19 vs VGG-16 Comparison vgg16_cfg = [64, 64, 'M', 128, 128, 'M', 256, 256, 256, 'M', 512, 512, 512, 'M', 512, 512, 512, 'M'] vgg19_cfg = [64, 64, 'M', 128, 128, 'M', 256, 256, 256, 256, 'M', 512, 512, 512, 512, 'M', 512, 512, 512, 512, 'M'] # ↑ extra ↑ extra ↑ extra # Performance Metrics: # VGG-16: 138M params, 15.5B FLOPs, 7.3% top-5 error # VGG-19: 144M params, 19.6B FLOPs, 7.3% top-5 error
Insight: Diminishing returns with depth - VGG-19 shows that simply adding more layers doesn't always improve performance significantly.

Architecture: FC-4096 → FC-4096 → FC-1000
# Classifier Implementation self.classifier = nn.Sequential( nn.Linear(512 * 7 * 7, 4096), # 25,088 → 4096 nn.ReLU(inplace=True), nn.Dropout(p=0.5), nn.Linear(4096, 4096), # 4096 → 4096 nn.ReLU(inplace=True), nn.Dropout(p=0.5), nn.Linear(4096, 1000) # 4096 → 1000 classes ) # Parameter count in FC layers: # FC1: 25,088 × 4,096 = 102,760,448 (74% of total!) # FC2: 4,096 × 4,096 = 16,777,216 # FC3: 4,096 × 1,000 = 4,096,000 # Total FC: 123,633,664 parameters

🎯 Architectural Simplicity

Design Principle: Use only 3×3 convolutions and 2×2 max pooling
# VGG Design Rules: 1. All conv layers: 3×3 filters, stride 1, pad 1 (preserves spatial size) 2. All pool layers: 2×2 max pool, stride 2 (halves spatial size) 3. Double channels after each pooling: 64 → 128 → 256 → 512 4. Stack multiple 3×3 convs instead of larger filters # Why this works: def receptive_field_comparison(): # One 5×5 conv: params_5x5 = 5 * 5 * C * C = 25C² # Two 3×3 convs: params_3x3 = 2 * (3 * 3 * C * C) = 18C² # Savings: 28% fewer parameters # Bonus: Extra non-linearity from 2 ReLUs vs 1
Benefits:
• Easy to implement and understand
• Uniform architecture throughout
• No hyperparameter tuning for filter sizes
• Influenced all future CNN designs
×

📊 Depth Matters

Key Finding: Deeper networks with small filters > Shallow networks with large filters
# Depth Experiment Results: Network Depth Top-5 Error VGG-11 11 10.4% VGG-13 13 9.9% VGG-16 16 8.1% VGG-19 19 8.0% # Comparison with AlexNet: AlexNet: 8 layers, 11×11, 5×5 filters → 15.3% error VGG-16: 16 layers, only 3×3 filters → 8.1% error # Why deeper is better: 1. More non-linear transformations 2. More abstract feature hierarchies 3. Better discrimination capability
Lesson Learned: Depth is crucial for representation learning, but there's a limit (VGG-19 barely improves over VGG-16)

This inspired: ResNet's solution to train even deeper networks (152 layers!)
×

🔍 Effective Receptive Fields

Key Insight: Stack of small filters = Large filter with benefits
# Receptive Field Calculations: # Single 7×7 conv: Parameters: 7 × 7 × C_in × C_out = 49C² Non-linearities: 1 # Three 3×3 convs: Parameters: 3 × (3 × 3 × C × C) = 27C² Non-linearities: 3 Savings: 45% fewer parameters! # Receptive field growth: Layer 1: 3×3 Layer 2: 3×3 → 5×5 (each pixel sees 5×5 of input) Layer 3: 3×3 → 7×7 (each pixel sees 7×7 of input) # Visual representation: Input: [x x x x x x x] Conv1: [ o o o ] 3×3 receptive field Conv2: [ O O O ] 5×5 receptive field Conv3: [ Θ ] 7×7 receptive field
Advantages of stacking:
• Fewer parameters (more efficient)
• More non-linearities (better learning)
• Better gradient flow
• Implicit regularization through decomposition

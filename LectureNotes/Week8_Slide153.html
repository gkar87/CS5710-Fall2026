Dropout Regularization

Core Concept

What is Dropout?
Randomly deactivating neurons during training to prevent overfitting
How it Works
Each neuron has probability p of being "dropped out" during training
Key Insight
Forces network to learn robust features that work with different neuron combinations
Common Dropout Rates
0.2 - 0.5
Click to explore different rates

Implementation

# Training phase def dropout_forward(x, dropout_rate=0.5, training=True): if training: mask = np.random.rand(*x.shape) > dropout_rate x = x * mask x = x / (1 - dropout_rate) # Scale up return x # Inference phase def dropout_inference(x): return x # No dropout during inference
# PyTorch implementation import torch.nn as nn model = nn.Sequential( nn.Linear(784, 256), nn.ReLU(), nn.Dropout(0.5), # 50% dropout nn.Linear(256, 128), nn.ReLU(), nn.Dropout(0.3), # 30% dropout nn.Linear(128, 10) )

Dropout Strategies

Standard Dropout
  • Random neuron deactivation
  • Uniform probability
  • Most common approach
  • Works well for fully connected layers
Spatial Dropout
  • Drops entire feature maps
  • Better for convolutional layers
  • Preserves spatial structure
  • Used in CNNs
DropConnect
  • Drops connections, not neurons
  • More fine-grained control
  • Computationally expensive
  • Better generalization
Variational Dropout
  • Learned dropout rates
  • Adaptive to data
  • Bayesian interpretation
  • Automatic sparsity

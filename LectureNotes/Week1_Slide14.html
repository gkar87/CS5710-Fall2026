Cross-Validation

What is Cross-Validation?

Cross-validation is a statistical method for evaluating machine learning models by training and testing them on different subsets of data to get a more reliable estimate of performance.
  • More robust performance estimates
  • Better use of limited data
  • Reduces overfitting to test set
  • Helps with model selection
  • Provides variance estimates

Problems It Solves

Traditional train-test splits have limitations that can lead to unreliable model evaluation and poor generalization.
  • Single test set may not be representative
  • Wastes valuable training data
  • Performance depends on random split
  • No sense of prediction uncertainty
  • Risk of overfitting to test set

Cross-Validation Methods

🔄 Hold-Out Validation
Split data once into train (70-80%) and test (20-30%) sets. Simple but limited.
✅ Pros:
• Fast and simple
• Good for large datasets
• Clear separation
❌ Cons:
• Wastes data
• High variance
• Split-dependent
📊 K-Fold Cross-Validation
Split data into k folds, train on k-1 folds, test on 1 fold. Repeat k times. Most popular method.
✅ Pros:
• Uses all data
• Reliable estimates
• Provides variance
❌ Cons:
• k times slower
• Still some randomness
• Memory intensive
🎯 Leave-One-Out CV
Use every single data point as test set once. k-fold where k = dataset size.
✅ Pros:
• Maximum data usage
• No randomness
• Unbiased estimates
❌ Cons:
• Very slow
• High variance
• Computationally expensive

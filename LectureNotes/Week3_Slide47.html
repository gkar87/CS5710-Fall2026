Gradient Descent for Logistic Regression

📈 Gradient Derivation

Gradient descent minimizes the logistic regression cost function by iteratively updating parameters in the direction of steepest descent.
Cost Function:
J(θ) = -1/m × Σ[y·log(σ(θᵀx)) + (1-y)·log(1-σ(θᵀx))]

Gradient:
∂J/∂θⱼ = 1/m × Σ(σ(θᵀx) - y)·xⱼ
  • Step 1: Derivative of Sigmoid
    σ'(z) = σ(z)(1-σ(z))
  • Step 2: Chain Rule Application
    ∂J/∂θⱼ = ∂J/∂σ × ∂σ/∂z × ∂z/∂θⱼ
  • Step 3: Simplification
    Result: (prediction - actual) × feature
  • Step 4: Vectorization
    ∇J = 1/m × Xᵀ(σ(Xθ) - y)

⚡ Update Algorithm

Gradient Descent Steps:

1. Initialize parameters θ randomly
2. Repeat until convergence:
   • Compute predictions: h = σ(Xθ)
   • Calculate gradient: ∇J = 1/m × Xᵀ(h - y)
   • Update: θ := θ - α∇J
   • Check convergence
Update Rule:

θⱼ := θⱼ - α × 1/m × Σ(hθ(xᵢ) - yᵢ)xᵢⱼ

Vectorized:
θ := θ - α/m × Xᵀ(σ(Xθ) - y)
Key Insight: The gradient has the same form as linear regression, but uses the sigmoid function for predictions!
Implementation Details
Python Implementation
def gradient_descent(X, y, alpha, iterations):
  m, n = X.shape
  theta = np.zeros(n)
  for i in range(iterations):
    h = sigmoid(X @ theta)
    gradient = X.T @ (h - y) / m
    theta -= alpha * gradient
  return theta
• Vectorized implementation
• Learning rate α controls step size
• Batch gradient descent
Convergence Tips
Learning Rate Selection:
• Too large: Divergence/oscillation
• Too small: Slow convergence
• Typical range: 0.01 to 1.0

Convergence Criteria:
• Cost change < threshold
• Gradient norm < threshold
• Maximum iterations reached

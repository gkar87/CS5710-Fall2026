🎯 Conv1: First Convolutional Layer

Layer Specifications: 96 filters of size 11×11×3 with stride 4
Why 11×11 filters?
Large receptive fields capture low-level features like edges and blobs more effectively in high-resolution images (224×224).
# PyTorch Implementation self.conv1 = nn.Conv2d(3, 96, kernel_size=11, stride=4, padding=2) self.relu1 = nn.ReLU(inplace=True) self.pool1 = nn.MaxPool2d(kernel_size=3, stride=2) # Output: 96 × 55 × 55 feature maps
Key Innovation: Split across 2 GPUs - 48 filters per GPU to fit in memory (3GB GTX 580)
×

🎯 Conv2: Second Convolutional Layer

Layer Specifications: 256 filters of size 5×5×48
This layer increases the depth significantly (96→256) to learn more complex features. The GPU split continues here with 128 filters per GPU.
# PyTorch Implementation self.conv2 = nn.Conv2d(96, 256, kernel_size=5, padding=2) self.relu2 = nn.ReLU(inplace=True) self.pool2 = nn.MaxPool2d(kernel_size=3, stride=2) # Output: 256 × 13 × 13 feature maps # Total parameters: 5×5×96×256 = 614,400
Cross-GPU Communication: This layer connects to both GPU streams from the previous layer, enabling richer feature combinations.
×

🎯 Conv3-5: Deep Convolutional Layers

Architecture:
• Conv3: 384 filters @ 3×3
• Conv4: 384 filters @ 3×3
• Conv5: 256 filters @ 3×3
# PyTorch Implementation self.conv3 = nn.Conv2d(256, 384, kernel_size=3, padding=1) self.relu3 = nn.ReLU(inplace=True) self.conv4 = nn.Conv2d(384, 384, kernel_size=3, padding=1) self.relu4 = nn.ReLU(inplace=True) self.conv5 = nn.Conv2d(384, 256, kernel_size=3, padding=1) self.relu5 = nn.ReLU(inplace=True) self.pool5 = nn.MaxPool2d(kernel_size=3, stride=2)
Design Choice: Smaller 3×3 filters in deeper layers capture high-level features while maintaining computational efficiency. No pooling between Conv3-4-5 preserves spatial information.

🔗 Fully Connected Layers

Architecture:
• FC6: 256×6×6 → 4096 neurons
• FC7: 4096 → 4096 neurons
• FC8: 4096 → 1000 classes
# PyTorch Implementation with Dropout self.dropout = nn.Dropout(p=0.5) self.fc6 = nn.Linear(256 * 6 * 6, 4096) self.relu6 = nn.ReLU(inplace=True) self.dropout6 = nn.Dropout(p=0.5) self.fc7 = nn.Linear(4096, 4096) self.relu7 = nn.ReLU(inplace=True) self.dropout7 = nn.Dropout(p=0.5) self.fc8 = nn.Linear(4096, 1000) # 1000 ImageNet classes
Parameter Count: FC layers contain ~90% of parameters (FC6: 37.7M, FC7: 16.8M, FC8: 4.1M)
Dropout: Critical for preventing overfitting in these parameter-heavy layers

ILSVRC 2012 Results:
• Top-5 Error: 15.3% (10.8% better than 2nd place)

Historical Context:
Before AlexNet, the best methods achieved ~26% top-5 error using hand-crafted features. AlexNet's dramatic improvement sparked the deep learning revolution.

⚡ ReLU Activation Function

Innovation: First large-scale use of ReLU in deep networks
# ReLU vs Traditional Activations def relu(x): return torch.max(torch.zeros_like(x), x) def sigmoid(x): return 1 / (1 + torch.exp(-x)) def tanh(x): return torch.tanh(x) # Training Speed Comparison: # ReLU: 6x faster convergence than tanh # No gradient vanishing in positive region # Sparse activation (typically 50% neurons are zero)
Benefits:
• Non-saturating gradients
• Computationally efficient (max(0, x))
• Introduces sparsity
• Faster convergence

Mathematical Form: f(x) = max(0, x)

🎲 Dropout Regularization

Key Innovation: First successful use of dropout in CNNs at scale
class Dropout(nn.Module): def __init__(self, p=0.5): super().__init__() self.p = p def forward(self, x): if self.training: # Randomly zero out neurons mask = torch.bernoulli(torch.ones_like(x) * (1 - self.p)) # Scale remaining neurons return x * mask / (1 - self.p) return x # AlexNet uses p=0.5 in FC6 and FC7 # Reduces overfitting significantly # Test error reduced by ~2% with dropout
Effect: Forces network to learn redundant representations, preventing co-adaptation of neurons. Equivalent to training an ensemble of 2^n sub-networks.

Training vs Inference: Applied only during training; all neurons active during inference with weights scaled by (1-p).
×

📊 Local Response Normalization

Concept: Implements lateral inhibition inspired by neuroscience
# Local Response Normalization Formula def local_response_norm(x, size=5, alpha=1e-4, beta=0.75, k=2): """ x: input tensor (N, C, H, W) Normalizes across neighboring channels """ n = size squared = x.pow(2) # Sum over neighboring channels pool = F.avg_pool3d( squared.unsqueeze(1), kernel_size=(n, 1, 1), stride=1, padding=(n//2, 0, 0) ).squeeze(1) # Normalization formula denominator = (k + alpha * pool).pow(beta) return x / denominator # AlexNet parameters: n=5, k=2, α=10^-4, β=0.75 # Applied after ReLU in conv1 and conv2
Effect: Creates competition among neurons at the same spatial location across different channels. Top-1 error reduced by 1.4%.

Note: Later research showed LRN less effective than batch normalization; rarely used in modern architectures.

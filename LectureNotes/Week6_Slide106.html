K-Means++ Initialization

The Problem with Random Init

Random initialization can lead to poor clustering results and slow convergence.

❌ Random Initialization

  • Centroids may be close together
  • Poor initial clusters
  • Slow convergence
  • Suboptimal results
  • High variance in outcomes

✅ K-Means++

  • Spreads initial centroids
  • Better starting clusters
  • Faster convergence
  • Improved final results
  • More consistent outcomes

K-Means++ Algorithm

Smart initialization that chooses centroids with probability proportional to their squared distance from existing centroids.
  1. 1 Choose first centroid randomly from data points
  2. 2 For each point, compute distance to nearest centroid
  3. 3 Choose next centroid with probability ∝ D(x)²
  4. 4 Repeat until K centroids are selected
  5. 5 Proceed with standard K-means algorithm

Implementation

def kmeans_plus_plus(X, K): # Step 1: Choose first centroid randomly centroids = [X[np.random.randint(X.shape[0])]] for _ in range(1, K): # Step 2: Calculate distances to nearest centroid distances = [] for x in X: min_dist = min([np.linalg.norm(x - c) for c in centroids]) distances.append(min_dist ** 2) # Step 3: Choose next centroid with weighted probability probabilities = distances / sum(distances) cumulative_probs = np.cumsum(probabilities) r = np.random.rand() for i, p in enumerate(cumulative_probs): if r < p: centroids.append(X[i]) break return np.array(centroids)

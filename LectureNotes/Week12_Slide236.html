REST APIs for ML Models

Framework Comparison

Modern, fast web framework for building APIs

  • Type hints and validation
  • Async support
  • High performance
  • Modern Python features
🌶️

Lightweight and flexible micro-framework

  • Flexible and customizable
  • Well-documented
  • Mature and stable
Quick Comparison:
FastAPI: Better for new projects, automatic docs, type safety
Flask: More mature ecosystem, greater flexibility, easier learning curve

FastAPI Example

from fastapi import FastAPI, HTTPException from pydantic import BaseModel import joblib import numpy as np # Initialize FastAPI app app = FastAPI(title="ML Model API") # Load trained model model = joblib.load("loan_model.pkl") # Request/Response schemas class PredictionRequest(BaseModel): credit_score: float annual_income: float debt_to_income: float employment_length: int class PredictionResponse(BaseModel): prediction: str probability: float confidence: str # Prediction endpoint @app.post("/predict", response_model=PredictionResponse) async def predict(request: PredictionRequest): try: features = np.array([[ request.credit_score, request.annual_income, request.debt_to_income, request.employment_length ]]) prediction = model.predict(features)[0] probability = model.predict_proba(features)[0].max() return PredictionResponse( prediction="approved" if prediction == 1 else "rejected", probability=probability, confidence="high" if probability > 0.8 else "low" ) except Exception as e: raise HTTPException(status_code=500, detail=str(e))

Common ML API Endpoints

POST
/predict
Make predictions on new data. Primary endpoint for inference requests.
curl -X POST /predict \ -H "Content-Type: application/json" \ -d '{"features": [0.5, 1.2, 0.8]}'
POST
/predict/batch
Process multiple predictions in a single request for efficiency.
curl -X POST /predict/batch \ -H "Content-Type: application/json" \ -d '{"samples": [[0.5,1.2], [0.8,0.3]]}'
GET
/model/info
Get model metadata, version, and feature information.
curl -X GET /model/info Response: {"version": "1.2.0", "features": ["age", "income"]}
GET
/health
Health check endpoint for monitoring and load balancer status.
curl -X GET /health Response: {"status": "healthy", "model_loaded": true}
POST
/explain
Get model explanations for individual predictions using SHAP or LIME.
curl -X POST /explain \ -H "Content-Type: application/json" \ -d '{"features": [0.5, 1.2], "method": "shap"}'
GET
/metrics
Expose metrics for monitoring system performance and model drift.
curl -X GET /metrics Response: {"requests_per_minute": 150, "avg_latency_ms": 45}

API Best Practices

🔒
Authentication & Security
Implement API keys, JWT tokens, or OAuth. Use HTTPS, validate inputs, and rate limit requests.
📋
Input Validation
Validate data types, ranges, and required fields. Use schema validation (Pydantic, Marshmallow).
⚡
Performance Optimization
Use async operations, batch processing, caching, and connection pooling for better throughput.
🚨
Error Handling
Return meaningful error codes, detailed messages, and consistent error response format.
📖
Documentation
Auto-generate API docs (OpenAPI/Swagger), provide examples, and maintain changelog.
📊
Monitoring & Logging
Log requests/responses, monitor latency, track error rates, and implement health checks.

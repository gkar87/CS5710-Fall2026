Tree Pruning Techniques

⚠️ The Overfitting Problem

Decision trees can grow very deep, memorizing training data rather than learning generalizable patterns. This leads to poor performance on new data.
Pruning is the process of removing branches from a decision tree to reduce complexity and improve generalization.

🎯 Why Prune?

  • • Reduce overfitting
  • • Improve generalization
  • • Create simpler, interpretable models
  • • Reduce computational cost
  • • Handle noise in training data

✂️ Pruning Methods

  • 🔄 Pre-pruning (Early Stopping)
    Stop growing the tree during construction based on stopping criteria.
    Examples: Maximum depth = 5, Minimum samples per leaf = 10, Minimum information gain = 0.01
  • ✂️ Post-pruning (Backward Pruning)
    Build a full tree first, then remove branches that don't improve validation performance.
    Build complete tree → Test each subtree removal → Keep changes that improve validation accuracy
  • 💰 Cost Complexity Pruning
    Balance tree accuracy against tree complexity using a complexity parameter (α).
    Cost = Error Rate + α × Tree Complexity. Higher α = more pruning

Batch Gradient Descent

What is Batch Gradient Descent?

Batch Gradient Descent uses the entire training dataset to compute the gradient at each iteration. It calculates the average error across all training examples before updating the model parameters.
θ = θ - α · ∇J(θ)

∇J(θ) = (1/m) Σᵢ₌₁ᵐ ∇ℓ(hθ(xⁱ), yⁱ)
Key Characteristics:

• Uses all m training examples per update
• Computes exact gradient direction
• Updates parameters once per epoch
• Guaranteed convergence to global minimum (convex functions)

Algorithm Steps

  • Step 1: Initialize Parameters
    Set initial values for weights θ and learning rate α
  • Step 2: Compute Predictions
    Calculate ŷⁱ = hθ(xⁱ) for all training examples
  • Step 3: Calculate Cost
    Compute J(θ) = (1/2m) Σ(ŷⁱ - yⁱ)²
  • Step 4: Compute Gradients
    Calculate ∂J/∂θⱼ for all parameters using entire dataset
  • Step 5: Update Parameters
    Update θⱼ = θⱼ - α · (∂J/∂θⱼ) for all j simultaneously
  • Step 6: Check Convergence
    Repeat until cost function converges or max iterations reached

🔍 Advantages vs Disadvantages

✅ Advantages
  • Stable convergence to global minimum
  • Smooth gradient descent path
  • Exact gradient computation
  • Well-suited for convex optimization
  • Efficient use of vectorized operations
  • Theoretical guarantees
❌ Disadvantages
  • Slow for large datasets
  • Memory intensive
  • Cannot handle streaming data
  • Expensive computation per update
  • May get stuck in local minima (non-convex)
  • Updates only after seeing all data
💡 When to Use Batch Gradient Descent:
Small to medium datasets (< 10,000 examples), when you need stable convergence, or when computational resources allow processing the entire dataset at once.

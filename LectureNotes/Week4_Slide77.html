SVM Hyperparameter Tuning

⚖️
C Parameter
Regularization Strength
Controls the trade-off between achieving low training error and low testing error. Higher C = less regularization.
Training Error:
Low C: Higher High C: Lower
Overfitting:
Low C: Less prone High C: More prone
Margin:
Low C: Wider High C: Narrower
🎯
Gamma (γ)
RBF Kernel Parameter
Defines how far the influence of a single training example reaches. Only relevant for RBF, polynomial, and sigmoid kernels.
Decision Boundary:
Low γ: Smooth High γ: Complex
Training Accuracy:
Low γ: Lower High γ: Higher
Generalization:
Low γ: Better High γ: Worse
🔧
Kernel Type
Feature Space Mapping
Determines how the algorithm maps input features to higher-dimensional space for non-linear separation.
Linear:
Fast, interpretable
RBF:
Most versatile
Polynomial:
Smooth boundaries
📈
Degree (d)
Polynomial Kernel Degree
Degree of polynomial kernel function. Only used when kernel='poly'. Higher degrees create more complex boundaries.
Complexity:
Low d: Simple High d: Complex
Training Time:
Low d: Fast High d: Slow
Typical Range:
2-4 (most common)

Hyperparameter Tuning Methods

🔍
Grid Search
Exhaustive search over specified parameter values. Systematic but can be slow for many parameters.
🎲
Random Search
Random sampling from parameter distributions. Often more efficient than grid search.
🧠
Bayesian Optimization
Uses prior results to intelligently choose next parameters to try. Most efficient but complex.

💡 Tuning Best Practices
  • Start with default parameters, then systematically tune
  • Use cross-validation to avoid overfitting to validation set
  • Try RBF kernel first - it works well for most problems
  • Scale features before tuning - SVM is sensitive to feature scales
  • Consider class imbalance with class_weight parameter
  • For large datasets, start with smaller C values

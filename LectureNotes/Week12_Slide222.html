Train-Validation-Test Split

The Golden Rule of ML

Never evaluate on the data you trained on! The train-validation-test split is the foundation of honest model evaluation and preventing overfitting.
Why Three Sets?

• Training Set: For learning model parameters
• Validation Set: For hyperparameter tuning and model selection
• Test Set: For final, unbiased performance evaluation

The Key Principle: Each set serves a specific purpose and must remain independent to ensure valid evaluation.

Split Strategies

  • 🎲 Random Split
    Standard approach for i.i.d. data
  • ⚖️ Stratified Split
    Maintains class distribution across sets
  • 📅 Temporal Split
    Respects time order for time series
  • 👥 Group-based Split
    Ensures groups don't leak across sets
  • 🗺️ Spatial Split
    For geographically distributed data

Typical Split Proportions

60-70%
Training Set
Model Learning
15-20%
Validation Set
Hyperparameter Tuning
15-20%
Test Set
Final Evaluation
